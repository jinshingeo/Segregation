## 1. RSG_PCstar

### 1-1. 개념

**Morgan의 PC* (Proximity Contact Star)** 는 1983년 Morgan이 제안한 공간 노출지수입니다.

기본 아이디어는 **"집단 A의 구성원이 주변에서 집단 B를 얼마나 만나는가"** 입니다.  
기존 노출지수(예: 비공간 P* 지수)는 각 구역의 인구 구성만 보고 공간적 위치를 전혀 고려하지 않습니다.  
Morgan은 여기에 **거리 감쇄 함수(distance decay function)** 를 도입해서, 가까운 구역일수록 더 큰 가중치를 주고 먼 구역은 적게 반영하도록 했습니다.

**핵심 수식 (거리 감쇄 함수):**

```
W_ij = 10^(a - b * d_ij^m)
```

- `d_ij` : 구역 i와 구역 j 사이의 거리
- `a` : 절편 (log10 스케일, 거리 0일 때의 접촉 강도)
- `b` : 기울기 (거리가 멀어질수록 감쇄 속도)
- `m` : 지수 (거리 변환 형태, 보통 0 < m ≤ 0.5)

이 가중치 행렬 W로 각 구역 i 주변의 집단B 비율을 계산하고, 집단A 인구 비중으로 가중 평균을 냅니다.

**왜 Gaussian이 아니라 이 함수인가?**

Gaussian 커널(다른 측도들이 쓰는 방식)은 수학적으로 단순하지만 실제 인간의 이동/접촉 패턴을 반드시 잘 반영하지는 않습니다. Morgan은 실제 관측된 접촉 데이터(거리별 접촉 빈도 C_obs)를 log 회귀로 피팅해서 a, b를 추정하도록 설계했습니다. 즉, **경험적으로 추정된 감쇄 함수** 를 쓰는 것이 이 측도의 차별점입니다.

### 1-2. 함수 설명

```r
RSG_PCstar(data, group_a, group_b, total,
           dist_matrix = NULL,
           a_param, b_param, m,
           verbose = FALSE)
```

#### 입력 인자

| 인자 | 타입 | 설명 |
|------|------|------|
| `data` | sf 객체 또는 NULL | 공간 데이터. sf이면 geometry에서 거리 자동 계산. NULL이면 `dist_matrix` 직접 입력 필요 |
| `group_a` | 문자열 또는 숫자벡터 | 집단 A (노출 주체, 소수집단). 열 이름("minority") 또는 숫자 벡터 직접 입력 가능 |
| `group_b` | 문자열 또는 숫자벡터 | 집단 B (노출 대상, 다수집단). 열 이름("majority") 또는 숫자 벡터 |
| `total` | 문자열 또는 숫자벡터 | 전체 인구. 열 이름("total") 또는 숫자 벡터 |
| `dist_matrix` | 행렬 또는 NULL | n×n 거리 행렬. NULL이면 data에서 자동 계산 |
| `a_param` | 숫자 | 감쇄 함수 절편 (log10 스케일). 실제 데이터로 회귀 추정하거나 논문 값 사용 |
| `b_param` | 숫자 | 감쇄 함수 기울기 (양수). 클수록 거리 증가에 따라 빠르게 감쇄 |
| `m` | 숫자 | 거리 변환 지수. 0 < m ≤ 0.5 권장. m=0.5이면 √d 변환 |
| `verbose` | 논리값 | TRUE이면 결과값 콘솔 출력 |

#### 함수 내부 흐름

```
1. .rsg_resolve_pop()  →  a_pop, b_pop, t_pop 벡터 추출
2. .rsg_validate_lengths()  →  세 벡터 길이 일치 확인, n 반환
3. .rsg_dist_matrix()  →  거리 행렬 계산 (또는 그대로 사용)
4. W = 10^(a_param - b_param * dist_mat^m)  ← 핵심 감쇄 행렬
5. L_b = W %*% b_pop  (구역i 주변의 집단B 가중 합계)
6. L_t = W %*% t_pop  (구역i 주변의 전체 인구 가중 합계)
7. local_prop = L_b / L_t  (구역i 주변의 집단B 비율)
8. pcstar = sum((a_pop / Total_A) * local_prop)  (집단A 인구 가중 평균)
```

#### 4번 단계가 핵심이자 이 측도만의 특징

다른 공간 노출지수들은 Gaussian 커널 `exp(-d²/2bw²)` 을 씁니다.  
PC*는 `10^(a - b*d^m)` 을 씁니다. 이 식은 **log10 선형 회귀로 a, b를 추정할 수 있게** 설계된 것입니다.

양변에 log10을 취하면:
```
log10(W_ij) = a - b * d_ij^m
```
이게 y = a + b'x 형태의 선형 회귀이므로, 실제 접촉 데이터가 있으면 `lm()`으로 바로 추정할 수 있습니다.

#### 예제에서 a, b, m을 어떻게 구하나?

`02_example_run_new.R`에서:

```r
d_mid <- c(0.5, 1.5, 2.5, 3.5, 4.5, 5.5, 6.5)   # 거리 구간 중점
C_obs <- c(2.032, 0.736, 0.568, 0.597, 0.386, 0.558, 0.471)  # 관측 접촉 빈도
m_fix <- 0.5   # m을 0.5로 고정 (√d 변환)

lm_fit <- lm(log10(C_obs) ~ I(d_mid^m_fix))
a_est  <- coef(lm_fit)[1]      # 절편
b_est  <- -coef(lm_fit)[2]     # 기울기 (부호 반전, 감쇄이므로 양수로)
```

실제 분석에서는 `C_obs` 자리에 실제 조사된 "거리별 집단 간 접촉 빈도" 데이터가 들어갑니다.  
예제에서는 임의로 설정한 값으로 회귀를 돌려 a, b를 추정한 것입니다.

#### 반환값

```r
list(index = pcstar_val)
```

- `index`: 0~1 사이의 단일 숫자. 집단A가 주변에서 만나는 집단B의 평균 비율. 높을수록 두 집단의 공간적 접촉이 많음(분리 낮음)

## 2. RSG_E_Activity

### 2-1. 개념

**활동공간 기반 노출지수 (Activity Space-Bounded Exposure Index)** 는 Wong & Shaw (2011)가 제안했습니다.

기존 공간 노출지수들은 **거주지 중심** 으로 분석합니다. 즉, "내가 사는 동네 주변에 얼마나 다른 집단이 있는가"를 봅니다. 하지만 현실에서 사람들은 집에만 있지 않고 직장, 학교, 마트 등으로 이동합니다. **실제로 어디에서 시간을 보내는지**가 더 정확한 노출을 반영합니다.

이 측도는 **OD(Origin-Destination) 행렬** — 즉 "어디서 어디로 이동하는가"의 실제 데이터 — 을 이용해서, 각 구역 사람들이 실제 방문하는 구역들의 인구 구성을 가중 평균합니다.

**핵심 아이디어:**

1. OD 행렬 φ[i,j] = 구역 i에 사는 사람이 구역 j로 이동한 수
2. 이것을 행 정규화(row-normalize)해서 가중치 행렬 W를 만든다
   - W[i,j] = φ[i,j] / Σ_j φ[i,j] (구역 i에서 j로 이동하는 비율)
   - 즉 W[i,j]는 "구역 i 주민이 구역 j에서 시간을 보내는 비율"
3. 각 구역의 집단B 비율 = b_pop / t_pop
4. 구역 i 주민이 경험하는 집단B 비율 = Σ_j W[i,j] * (b_j / t_j)
5. 전역 노출지수 = Σ_i (a_i / Total_A) * (구역i 주민이 경험하는 집단B 비율)

**왜 이 측도가 중요한가?**

거주지 기반 노출지수는 실제 생활 반경을 무시합니다. 예를 들어 빈부가 극도로 분리된 도시라도 직장이 혼합된 지역에 있다면 실제 만남은 많을 수 있습니다. 반대로 거주지는 섞여있어도 각자 다른 지역으로 출퇴근하면 실제 노출은 낮을 수 있습니다.

### 2-2. 함수 설명

```r
RSG_E_Activity(data = NULL, group_a, group_b, total,
               od_matrix,
               verbose = FALSE)
```

#### 입력 인자

| 인자 | 타입 | 설명 |
|------|------|------|
| `data` | sf 객체 또는 NULL | 공간 데이터. 이 함수에서 공간 연산은 하지 않으므로 사실상 열 이름 파싱용. NULL도 가능 |
| `group_a` | 문자열 또는 숫자벡터 | 집단 A (노출 주체). 열 이름 또는 숫자 벡터 |
| `group_b` | 문자열 또는 숫자벡터 | 집단 B (노출 대상). 열 이름 또는 숫자 벡터 |
| `total` | 문자열 또는 숫자벡터 | 전체 인구. 열 이름 또는 숫자 벡터 |
| `od_matrix` | n×n 행렬 | **핵심 입력**. od_matrix[i,j] = 구역 i → 구역 j 이동 인원수. 양수여야 함 |
| `verbose` | 논리값 | TRUE이면 결과값 콘솔 출력 |

**`od_matrix` 주의사항:**
- 반드시 n×n 정방행렬 (n = 구역 수)
- 음수 불가
- 대각(i=j)은 자기 구역 내 머무는 인원 — 함수 내부에서 0으로 처리함

#### 함수 내부 흐름

```
1. .rsg_resolve_pop()  →  a_pop, b_pop, t_pop 벡터 추출
2. .rsg_validate_lengths()  →  벡터 길이 확인 → n
3. .rsg_validate_od()  →  od_matrix 크기(n×n) 및 음수 검증
4. .rsg_normalize_od()  →  대각 0으로 설정 후 행 합으로 나눠서 W 행렬 생성
5. b_ratio = b_pop / t_pop  (각 구역의 집단B 비율)
6. local_exp = W %*% b_ratio  (구역i 주민이 경험하는 집단B 비율)
7. global_exp = sum((a_pop / Total_A) * local_exp)  (집단A 전체의 평균 노출)
```

#### 4번 단계: `.rsg_normalize_od()` 상세

```r
.rsg_normalize_od <- function(phi) {
  diag(phi) <- 0           # 자기 구역 제거
  row_sums  <- rowSums(phi)
  zero_rows <- row_sums == 0
  # 유출량이 0인 구역은 균등 가중치로 대체 (경고 발생)
  phi / row_sums
}
```

대각을 0으로 만드는 이유: OD 행렬의 대각은 "구역 내 이동"이지만, 이 측도에서는 **다른 구역으로의 이동만** 고려하기 때문입니다. 대각을 남겨두면 "내 구역 안에서 나 자신을 만나는 비율"이 들어가서 노출 개념이 왜곡됩니다.

유출량이 0인 구역(아무 데도 이동 없음)은 균등 가중치 (1/n씩)로 대체하고 경고를 띄웁니다. 이는 수치 오류 방지용 안전장치입니다.

#### 반환값

```r
list(index = global_exp, local = local_exp)
```

- `index`: 전역 활동공간 노출지수 (단일 숫자)
- `local`: 구역별 기여값 벡터 (길이 n) — 각 구역이 전역 index에 얼마나 기여하는지

## 3. RSG_FSxPy

### 3-1. 개념

**흐름 기반 공간 노출지수 (Flow-based Spatial Exposure Index)** 는 Liu, Liu & Ye (2021)이 제안했습니다.

이름의 FSxPy는 **"Flow Space exposure of group x to group y"** 의 약자입니다.

`RSG_E_Activity`와 마찬가지로 OD 행렬을 사용하지만, 훨씬 정교합니다.  
`RSG_E_Activity`의 한계: OD 행렬을 단순히 행 정규화하면, 이동량이 많은 사람(활동적인 구역)과 적은 사람(비활동적인 구역)을 동일하게 취급합니다.

FSxPy의 핵심 혁신은 두 가지입니다:

**혁신 1: 자기 구역 가중치 포함 (Self-mixing weight)**

실제로 사람들은 자기 구역에서도 시간을 씁니다. 단순히 "다른 구역으로 이동한 비율"만 보면 집에서 보내는 시간이 무시됩니다. FSxPy는 **자기 구역에 머무는 비율(α)을 명시적으로 설정** 합니다.

**혁신 2: 계층(Hierarchy) 구조 — 활동성에 따른 자기혼합비율 차등화**

이동량(유출량)이 많은 구역은 더 "활동적"입니다. 활동적일수록 자기 구역에 머무는 비율이 낮아야 합니다(더 많이 돌아다니니까). 반대로 이동량이 적은 구역은 자기 구역에 더 많이 머뭅니다.

이를 위해 Lorenz 곡선으로 구역들을 **계층(h)**으로 분류합니다:
- h=1: 가장 이동량이 많은 구역 (자기혼합비율 가장 낮음 = α)
- h=max_level: 가장 이동량이 적은 구역 (자기혼합비율 가장 높음 = β)

**ω 가중치 행렬 구성:**

```
ω[i,i] = α + (h_i - 1) * (β - α) / (max_level - 1)
ω[i,j] = (OD 기반 비율) × (1 - ω[i,i])  (i≠j)
```

즉, 자기 구역 체류 비율 ω[i,i]를 먼저 계층에 따라 정하고, 나머지 (1-ω[i,i])를 OD 비율에 비례해서 분배합니다.

---

### 3-2. 함수 설명

```r
RSG_FSxPy(data = NULL, group_a, group_b, total,
          od_matrix,
          alpha = 0.3, beta = 1.0, max_level = 6L,
          hierarchy = NULL,
          verbose = FALSE)
```

#### 입력 인자

| 인자 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `data` | sf 또는 NULL | NULL | 공간 데이터 (열 이름 파싱용) |
| `group_a` | 문자열 또는 숫자벡터 | — | 집단 A (노출 주체). `group_x`에 해당 |
| `group_b` | 문자열 또는 숫자벡터 | — | 집단 B (노출 대상). `group_y`에 해당 |
| `total` | 문자열 또는 숫자벡터 | — | 전체 인구 |
| `od_matrix` | n×n 행렬 | — | OD 이동량 행렬. [i,j] = 구역i→구역j 이동 수 |
| `alpha` | 숫자 | 0.3 | 최상위 활동 계층(h=1)의 자기혼합비율 하한. 0 ≤ α ≤ β ≤ 1 |
| `beta` | 숫자 | 1.0 | 최하위 계층(h=max_level)의 자기혼합비율 상한 |
| `max_level` | 정수 | 6L | Lorenz 계층 분류 최대 단계 수 (2 이상) |
| `hierarchy` | 정수벡터 또는 NULL | NULL | 사전 계산된 계층 벡터. NULL이면 자동 계산 |
| `verbose` | 논리값 | FALSE | TRUE이면 결과값 콘솔 출력 |

**alpha, beta 해석:**
- `alpha = 0.3`: 가장 활동적인 구역(많이 돌아다니는)도 자기 구역에서 최소 30%는 시간을 보낸다
- `beta = 1.0`: 가장 비활동적인 구역은 100% 자기 구역에 머문다 (이론적 상한)
- 논문 기본값이 0.3, 1.0이고, 실제 통근 데이터가 있으면 조정 가능

#### 함수 내부 흐름

```
1. .rsg_resolve_pop()  →  pop_x, pop_y, pop_t 벡터 추출
2. .rsg_validate_lengths()  →  n 확인
3. .rsg_validate_od()  →  phi 행렬 검증
4. alpha, beta, max_level 유효성 검사
5. hierarchy 계산 (NULL이면 .rsg_lorenz_hierarchy() 호출)
6. .rsg_build_omega()  →  ω 가중치 행렬 구성
7. mixed_y = ω %*% pop_y  (구역i 주민이 경험하는 집단B 인구)
8. mixed_t = ω %*% pop_t  (구역i 주민이 경험하는 전체 인구)
9. local_v = (pop_x / Total_X) * (mixed_y / mixed_t)
10. fsxpy_val = sum(local_v)
```

#### 5번 단계: `.rsg_lorenz_hierarchy()` 상세

Lorenz 곡선으로 구역을 계층 분류하는 내부 함수입니다.

```r
.rsg_lorenz_hierarchy <- function(outflow_totals, max_level = 6L)
```

- `outflow_totals`: 각 구역의 총 유출량 (대각 제외한 OD 행 합계)
- 알고리즘: Lorenz 곡선의 마지막 구간 기울기(slope)가 1 이하가 될 때까지 상위 집단을 계층 1, 2, ... 로 분류
- 기울기 > 1: "이 구역들은 전체 이동량에서 불균형적으로 큰 비중을 차지함" → 상위 계층
- 기울기 ≤ 1: "나머지는 비교적 균등함" → 현재 level로 통합

**쉬운 비유:** 소득 불평등을 측정하는 Lorenz 곡선과 동일한 원리. 상위 20%가 80%의 이동을 담당하는 구조를 감지해서 계층화합니다.

#### 6번 단계: `.rsg_build_omega()` 상세

```r
.rsg_build_omega <- function(phi, h, alpha, beta, max_level)
```

1. OD 행렬에서 대각 제거, 행 정규화 → 기본 비율 행렬
2. 각 구역의 자기혼합비율 계산:
   ```
   ω[i,i] = alpha + (h[i] - 1) * (beta - alpha) / (max_level - 1)
   ```
   - h=1이면: α + 0 = α (최소 자기혼합)
   - h=max_level이면: α + (max_level-1)*(β-α)/(max_level-1) = β (최대 자기혼합)
3. 비대각 원소: OD 비율 × (1 - ω[i,i])로 재조정

#### 반환값

```r
list(index = fsxpy_val, local = local_v)
```

- `index`: 전역 흐름 기반 노출지수 (단일 숫자)
- `local`: 구역별 기여값 벡터 (길이 n)

## 4. OD 행렬 공통 설명

### 4-1. OD 행렬이란?

OD 행렬(Origin-Destination Matrix)은 **"출발지 → 목적지"의 이동 흐름** 을 나타내는 n×n 행렬입니다.

```
       목적지1  목적지2  목적지3
출발지1 [  0       15       8   ]
출발지2 [ 12        0      22   ]
출발지3 [  5       18       0   ]
```

- od_matrix[1,2] = 15: 구역1에서 구역2로 이동한 사람이 15명
- 대각(self-loop): 자기 구역 내 이동 (RSG에서는 보통 0으로 처리)

### 4-2. 이 측도들이 특수 상황에만 작동하는 게 아닌가?

**아닙니다.** 이 함수들은 **OD 데이터가 있는 모든 상황에서 범용적으로 작동** 합니다.

입력 형식만 맞으면 됩니다:
- n×n 행렬
- [i,j] 원소 = 구역i → 구역j 이동량 (양수)
- 단위: 명, 통행 수, 비율 모두 가능 (내부에서 행 정규화하므로)

어떤 종류의 이동 데이터든 OD 행렬 형태로 만들 수 있으면 사용 가능합니다:
- 통근 데이터 (집→직장)
- 쇼핑 이동 데이터
- 학교 통학 데이터
- 교통카드 데이터
- 휴대폰 위치 데이터
- 설문 조사 이동 데이터

### 4-3. 교수님이 주신 예제 데이터(segdata.rda)에 OD 데이터가 있는가?

**없습니다.** `segdata.rda`에는 오직 구역별 **인구 구성(minority, majority 인구수)** 만 있고, OD 이동 데이터는 포함되어 있지 않습니다.

### 4-4. 그럼 예제에서 OD 행렬을 어떻게 만들었나?

`02_example_run_new.R`에서 **거리 기반으로 인공 합성한 OD 행렬** 을 사용합니다:

```r
od_mat <- exp(-dist_mat * 0.5)   # 거리가 멀수록 이동량이 지수적으로 감소
diag(od_mat) <- 0                 # 자기 구역 이동 제거
```

이것은 "멀수록 이동이 적다"는 직관적 가정으로 만든 **시뮬레이션 OD 행렬** 입니다.  
실제 OD 데이터가 없을 때 함수 테스트를 위한 더미(proxy) 데이터입니다.

**교수님께 설명할 포인트:**
> "교수님이 주신 segdata는 인구 구성 데이터만 있어서 OD 행렬이 없었습니다. OD 기반 측도를 실행하기 위해 거리 감쇄 함수(exp(-d*0.5))로 이동 가능성을 추정한 합성 OD 행렬을 생성했습니다. 실제 통근/이동 데이터가 있다면 그 행렬을 od_matrix에 그대로 넣으면 됩니다."

### 4-5. 실제 OD 데이터가 있을 때 어떻게 쓰는가?

예시: 국토교통부 가구통행실태조사, 서울 교통카드 데이터, 통계청 통근통학 OD 등

```r
# 예: CSV 파일로 된 OD 데이터를 행렬로 변환
od_raw <- read.csv("od_commute_2020.csv")
# od_raw: from_zone, to_zone, count 형태

od_matrix <- matrix(0, nrow = n_zones, ncol = n_zones)
for (k in seq_len(nrow(od_raw))) {
  i <- od_raw$from_zone[k]
  j <- od_raw$to_zone[k]
  od_matrix[i, j] <- od_raw$count[k]
}

# 이걸 그대로 함수에 입력
result <- RSG_E_Activity(
  data      = my_sf,
  group_a   = "minority",
  group_b   = "majority",
  total     = "total",
  od_matrix = od_matrix,
  verbose   = TRUE
)
```

### 4-6. E_Activity vs FSxPy — 어떤 차이가 있고 언제 쓰나?

| 항목 | RSG_E_Activity | RSG_FSxPy |
|------|----------------|-----------|
| OD 처리 | 단순 행 정규화 (대각 제거) | 계층별 자기혼합비율 + OD 비율 결합 |
| 자기 구역 체류 | 무시 (0 처리) | 명시적 반영 (α~β 범위) |
| 구역 활동성 차등화 | 없음 | Lorenz 계층으로 차등화 |
| 계산 복잡도 | 낮음 | 높음 |
| 필요 파라미터 | od_matrix만 | od_matrix + alpha + beta + max_level |
| 적합한 상황 | OD 데이터만 있고 단순 분석 | 구역별 활동 패턴 차이가 클 때 정밀 분석 |

둘 다 OD 행렬을 쓰지만, FSxPy가 이론적으로 더 정교합니다. 실제 적용에서는 데이터 품질과 분석 목적에 따라 선택합니다.
