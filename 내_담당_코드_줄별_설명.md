## 1. RSG_PCstar.R

### 전체 코드

```r
RSG_PCstar <- function(data = NULL, group_a, group_b, total, dist_matrix = NULL,
                       a_param, b_param, m, verbose = FALSE) {

  a_pop <- .rsg_resolve_pop(data, group_a)   # 23번줄
  b_pop <- .rsg_resolve_pop(data, group_b)   # 24번줄
  t_pop <- .rsg_resolve_pop(data, total)     # 25번줄
  n     <- .rsg_validate_lengths(a_pop, b_pop, t_pop)  # 26번줄

  Total_A <- sum(a_pop)                      # 28번줄
  if (Total_A == 0) return(invisible(list(index = NA_real_, summary = "기준 집단 인구 0")))  # 29번줄

  d_mat <- .rsg_dist_matrix(data, dist_matrix, n)      # 31번줄
  W     <- 10^(a_param - b_param * (d_mat^m))          # 32번줄

  L_b <- .rsg_local_intensity(W, b_pop)     # 34번줄
  L_t <- .rsg_local_intensity(W, t_pop)     # 35번줄

  local_prop <- ifelse(L_t > 0, L_b / L_t, 0)          # 37번줄
  pcstar_val <- sum((a_pop / Total_A) * local_prop, na.rm = TRUE)  # 38번줄

  if (verbose) cat(sprintf("Morgan's PC*: %.4f\n", pcstar_val))    # 40번줄

  invisible(list(index = pcstar_val))        # 42번줄
}
```

### 줄별 설명

#### 함수 선언부

```r
RSG_PCstar <- function(data = NULL, group_a, group_b, total, dist_matrix = NULL,
                       a_param, b_param, m, verbose = FALSE)
```

- `data = NULL`: 기본값이 NULL이라는 뜻. 안 넣어도 함수가 작동함. sf 객체를 넣으면 그걸 쓰고, 안 넣으면 숫자벡터 직접 입력으로 처리.
- `dist_matrix = NULL`: 거리 행렬도 직접 안 넣으면 `data`에서 자동 계산.
- `a_param, b_param, m`: 기본값이 없음 → **반드시 입력해야 하는 필수 인자**. 안 넣으면 에러남.
- `verbose = FALSE`: 기본은 조용히 실행. `TRUE`로 바꾸면 콘솔에 결과 출력.

---

#### 입력 파싱 블록 (23~26번줄)

```r
a_pop <- .rsg_resolve_pop(data, group_a)
b_pop <- .rsg_resolve_pop(data, group_b)
t_pop <- .rsg_resolve_pop(data, total)
n     <- .rsg_validate_lengths(a_pop, b_pop, t_pop)
```

**왜 이 4줄이 항상 맨 처음 나오나?**

사용자가 `group_a = "minority"` (열 이름)로 넣든 `group_a = c(100, 200, ...)` (숫자벡터)로 넣든,
이 아래 계산 코드는 **무조건 숫자벡터만 다룰 수 있어야** 합니다.
`.rsg_resolve_pop()`이 두 형식을 모두 숫자벡터로 통일해줍니다.

`.rsg_validate_lengths()`는 세 벡터의 길이가 모두 같은지 확인하고 n(구역 수)을 반환합니다. 여기서 걸러지지 않으면 나중에 행렬 연산에서 알기 어려운 오류가 납니다.

---

#### 조기 종료 (28~29번줄)

```r
Total_A <- sum(a_pop)
if (Total_A == 0) return(invisible(list(index = NA_real_, summary = "기준 집단 인구 0")))
```

**왜 `Total_A == 0` 을 먼저 체크하나?**

집단A 전체 인구가 0이면 아래 38번줄에서 `a_pop / Total_A`를 계산할 때 **0으로 나누는** 상황이 생깁니다. R은 이 경우 오류 대신 `NaN`이나 `Inf`를 반환해서 나중에 결과가 이상하게 나옵니다. 조기에 체크하고 `NA`를 반환하는 게 훨씬 안전합니다.

**`NA_real_`이 뭔가?**

그냥 `NA`와 같은데 타입이 실수형(double)임을 명시한 것입니다. `list(index = NA_real_)`로 반환하면 나중에 `result$index`를 숫자처럼 다룰 수 있습니다.

**`invisible()`이 왜 붙어있나?**

R에서 함수가 값을 반환할 때 콘솔에 자동으로 출력됩니다. `invisible()`로 감싸면 **값은 반환하되 콘솔 출력은 안 됩니다**. 사용자가 `result <- RSG_PCstar(...)` 처럼 변수에 저장하면 정상 사용 가능하고, 콘솔이 지저분해지지 않습니다.

---

#### 핵심 계산: 거리 감쇄 행렬 (31~32번줄)

```r
d_mat <- .rsg_dist_matrix(data, dist_matrix, n)
W     <- 10^(a_param - b_param * (d_mat^m))
```

**이 두 줄이 이 함수의 핵심입니다.**

`d_mat`은 n×n 행렬. `d_mat[i,j]` = 구역i와 구역j 사이의 거리.

`W`도 n×n 행렬. `W[i,j]` = 구역i와 구역j 사이의 **접촉 강도(가중치)**.

**수식: `W[i,j] = 10^(a - b × d[i,j]^m)`**

- 거리가 0이면: `10^(a - 0) = 10^a` → 가장 큰 가중치
- 거리가 커질수록: `b × d^m`이 커져서 지수가 음수 방향으로 감소 → 가중치 감소
- `d_mat^m`: R에서 행렬에 `^m`을 하면 **원소별로** 거듭제곱 적용. 행렬 전체에 한번에 계산됨.
- `10^(...)`: 역시 원소별 계산. for 루프 없이 n×n 전체를 한번에 처리.

**왜 for 루프 안 쓰나?**

```r
# 이렇게 짜도 되지만
for (i in 1:n) {
  for (j in 1:n) {
    W[i,j] <- 10^(a_param - b_param * d_mat[i,j]^m)
  }
}
# 이게 훨씬 빠르고 간결
W <- 10^(a_param - b_param * (d_mat^m))
```

R은 벡터/행렬 연산이 내부적으로 C로 최적화되어 있어서 for 루프보다 수십 배 빠릅니다. 구역이 100개면 10,000번 연산인데, 행렬 방식은 한 번에 처리합니다.

---

#### 국지 강도 계산 (34~35번줄)

```r
L_b <- .rsg_local_intensity(W, b_pop)
L_t <- .rsg_local_intensity(W, t_pop)
```

내부적으로는 `W %*% b_pop`와 같습니다.

- `W %*% b_pop`: 행렬 곱. 결과는 길이 n의 벡터.
- `L_b[i]` = `Σ_j W[i,j] × b_pop[j]` = 구역i 주변의 집단B 가중합
- `L_t[i]` = `Σ_j W[i,j] × t_pop[j]` = 구역i 주변의 전체인구 가중합

쉽게 말하면: "구역i 입장에서 주변을 둘러봤을 때 집단B가 얼마나 있나"를 거리 가중치 반영해서 계산한 값입니다.

---

#### 비율 계산 및 조기 분기 (37번줄)

```r
local_prop <- ifelse(L_t > 0, L_b / L_t, 0)
```

**`ifelse()`가 뭔가?**

`if/else`의 벡터 버전입니다.

```r
# if/else는 값 하나에만 작동
if (L_t[1] > 0) L_b[1] / L_t[1] else 0

# ifelse()는 벡터 전체에 한번에 작동
ifelse(L_t > 0, L_b / L_t, 0)
# → L_t[i] > 0이면 L_b[i]/L_t[i], 아니면 0 — 100개 구역 한번에 처리
```

**왜 0으로 나누는 걸 체크하나?**

`L_t[i]`가 0이 되는 경우: 구역i 주변에 아무도 없는 경우(인구 0인 구역). 이때 `L_b / L_t`는 `0/0 = NaN`이 되므로 방어 코드가 필요합니다.

---

#### 전역 지수 계산 (38번줄)

```r
pcstar_val <- sum((a_pop / Total_A) * local_prop, na.rm = TRUE)
```

- `a_pop / Total_A`: 집단A 전체에서 각 구역의 비중. 합산하면 1.
- `* local_prop`: 각 구역의 "주변 집단B 비율"에 집단A 비중 가중치를 곱함.
- `sum(...)`: 가중평균. "집단A 전체 입장에서 평균적으로 주변에 집단B가 얼마나 있나".
- `na.rm = TRUE`: 혹시 NaN이 남아있어도 무시하고 합산. 방어 코드.

**수식으로 표현:**
```
PC* = Σ_i (a_i / Total_A) × (L_b[i] / L_t[i])
```

---

#### 출력 및 반환 (40~42번줄)

```r
if (verbose) cat(sprintf("Morgan's PC*: %.4f\n", pcstar_val))
invisible(list(index = pcstar_val))
```

- `sprintf("%.4f", pcstar_val)`: 소수점 4자리로 포맷. `0.3421` 같은 형태로 출력.
- `\n`: 줄바꿈.
- `list(index = pcstar_val)`: 반환값을 list로 감싸는 이유 — 나중에 다른 값도 추가하기 쉽고, `result$index`로 접근하는 일관된 인터페이스를 유지하기 위해서.

## 2. RSG_E_Activity.R

### 전체 코드

```r
RSG_E_Activity <- function(data = NULL, group_a, group_b, total,
                           od_matrix, verbose = FALSE) {

  a_pop <- .rsg_resolve_pop(data, group_a)   # 20번줄
  b_pop <- .rsg_resolve_pop(data, group_b)   # 21번줄
  t_pop <- .rsg_resolve_pop(data, total)     # 22번줄
  n     <- .rsg_validate_lengths(a_pop, b_pop, t_pop)  # 23번줄

  Total_A <- sum(a_pop, na.rm = TRUE)        # 25번줄
  if (Total_A == 0) return(invisible(list(index = NA_real_, summary = "기준 집단 인구 0")))  # 26번줄

  phi <- .rsg_validate_od(od_matrix, n)      # 28번줄
  W   <- .rsg_normalize_od(phi)              # 29번줄

  b_ratio   <- ifelse(t_pop > 0, b_pop / t_pop, 0)         # 31번줄
  local_exp <- .rsg_local_intensity(W, b_ratio)             # 32번줄
  global_exp <- sum((a_pop / Total_A) * local_exp, na.rm = TRUE)  # 33번줄

  if (verbose) cat(sprintf("Activity Space Exposure: %.4f\n", global_exp))  # 35번줄

  invisible(list(index = global_exp, local = local_exp))    # 37번줄
}
```

---

### 줄별 설명

#### 입력 파싱 블록 (20~26번줄)

PCstar와 동일한 패턴. 차이점은 `na.rm = TRUE`가 `sum(a_pop)`에 명시적으로 붙어있는 것.

```r
Total_A <- sum(a_pop, na.rm = TRUE)
```

인구 데이터에 NA가 섞여있을 수 있으므로 NA를 제외하고 합산합니다. PCstar와 일관성이 약간 다르지만 방어적 코딩입니다.

---

#### OD 행렬 처리 (28~29번줄)

```r
phi <- .rsg_validate_od(od_matrix, n)
W   <- .rsg_normalize_od(phi)
```

**28번줄 — 검증:**

- `od_matrix`가 n×n인지, 음수는 없는지 확인.
- `phi`라는 이름으로 저장하는 이유: 논문(Wong & Shaw 2011)에서 OD 행렬을 φ(phi)로 표기함. 수식과 코드를 일치시킨 것.

**29번줄 — 정규화:**

`.rsg_normalize_od()` 내부에서 두 가지를 합니다:

```r
diag(phi) <- 0           # ① 대각 제거 (자기 구역 이동 무시)
phi / rowSums(phi)       # ② 행 정규화 (각 행 합 = 1)
```

① 왜 대각을 0으로 만드나?

OD 행렬에서 `phi[i,i]`는 구역i 내부 이동(자기 구역에서 자기 구역으로). 이 측도에서는 "다른 구역 사람을 얼마나 만나는가"를 보기 때문에 자기 구역 내 머묾은 제외합니다.

② 왜 행 정규화하나?

정규화 전: `phi[i,j]` = 구역i에서 구역j로 이동한 절대 인원수  
정규화 후: `W[i,j]` = 구역i 유출량 중 구역j로 가는 비율 (0~1)

이 비율이 "구역i 주민이 구역j에서 시간을 보내는 비중"으로 해석됩니다.

---

#### 핵심 계산 (31~33번줄)

```r
b_ratio   <- ifelse(t_pop > 0, b_pop / t_pop, 0)
local_exp <- .rsg_local_intensity(W, b_ratio)
global_exp <- sum((a_pop / Total_A) * local_exp, na.rm = TRUE)
```

**31번줄 — 집단B 비율:**

```r
b_ratio[j] = b_pop[j] / t_pop[j]
           = 구역j의 집단B 비율 (0~1)
```

`ifelse(t_pop > 0, ...)`: 인구 0인 구역은 비율 계산 불가이므로 0으로 처리.

**32번줄 — 활동공간 내 집단B 비율:**

내부적으로 `W %*% b_ratio` 행렬 곱.

```
local_exp[i] = Σ_j W[i,j] × b_ratio[j]
             = 구역i 주민이 이동할 때 만나는 집단B의 평균 비율
```

풀어쓰면: "구역i에서 출발하는 사람이 구역j에서 보내는 시간 비율(W[i,j])과 구역j의 집단B 비율(b_ratio[j])을 곱한 걸 모든 j에 대해 더함"

**33번줄 — 전역 지수:**

```
global_exp = Σ_i (a_pop[i] / Total_A) × local_exp[i]
           = 집단A 전체 입장에서의 가중 평균 노출
```

집단A 인구가 많은 구역의 `local_exp`에 더 큰 비중을 줍니다.

---

#### 반환값 (37번줄)

```r
invisible(list(index = global_exp, local = local_exp))
```

PCstar와 달리 **`local`도 같이 반환**합니다.

- `index`: 전역 노출지수 (단일 숫자)
- `local`: 구역별 벡터 (길이 n) — `local_exp[i]` = 구역i 주민이 경험하는 집단B 비율

**주의:** `sum(local) ≠ index`. `local`은 인구 가중치가 아직 안 곱해진 raw값입니다.  
진짜 기여값은 `(a_pop / Total_A) * local`을 계산해야 합니다.

## 3. RSG_FSxPy.R

이 파일은 내부 헬퍼 함수 2개 + 메인 함수 1개로 구성됩니다.

### 3-1. `.rsg_lorenz_hierarchy()` — Lorenz 계층 분류

#### 전체 코드

```r
.rsg_lorenz_hierarchy <- function(outflow_totals, max_level = 6L) {
  n         <- length(outflow_totals)   # 24번줄
  h         <- rep(max_level, n)        # 25번줄
  remaining <- seq_len(n)               # 26번줄
  level     <- 1L                       # 27번줄

  while (level < max_level && length(remaining) > 1L) {  # 29번줄
    flows <- outflow_totals[remaining]  # 30번줄
    ord   <- order(flows)               # 31번줄
    cum_n <- seq_along(ord) / length(ord)          # 32번줄
    cum_f <- cumsum(flows[ord]) / sum(flows[ord])  # 33번줄

    nr    <- length(ord)                # 35번줄
    slope <- (cum_f[nr] - cum_f[nr - 1L]) / (cum_n[nr] - cum_n[nr - 1L])  # 36번줄
    if (slope <= 1) break               # 37번줄

    x_thr     <- 1 - 1 / slope         # 39번줄
    cum_rank  <- rank(flows, ties.method = "first") / length(flows)  # 40번줄
    hot_local <- which(cum_rank > x_thr)  # 41번줄
    if (length(hot_local) == 0L) break    # 42번줄

    h[remaining[hot_local]] <- level   # 44번줄
    remaining <- remaining[-hot_local] # 45번줄
    level     <- level + 1L            # 46번줄
  }
  h
}
```

#### 줄별 설명

**초기화 (24~27번줄)**

```r
n         <- length(outflow_totals)  # 구역 수
h         <- rep(max_level, n)       # 모든 구역을 일단 최하위 계층으로 초기화
remaining <- seq_len(n)              # 아직 분류 안 된 구역 인덱스 (처음엔 전체)
level     <- 1L                      # 현재 분류 중인 계층 (1부터 시작)
```

`rep(max_level, n)`: 길이 n짜리 벡터를 만들고 모든 원소를 `max_level`로 채움.  
예: `rep(6, 5)` → `c(6, 6, 6, 6, 6)`. 일단 전부 최하위(6)로 놓고, 이후 반복에서 상위 구역들을 작은 숫자로 교체하는 방식.

`seq_len(n)`: `c(1, 2, 3, ..., n)`. `1:n`과 같지만 n=0일 때 안전합니다.

**반복 조건 (29번줄)**

```r
while (level < max_level && length(remaining) > 1L)
```

두 조건 중 하나라도 거짓이면 루프 종료:
- `level < max_level`: 최대 계층에 도달하면 종료
- `length(remaining) > 1L`: 분류할 구역이 1개 이하면 종료 (혼자는 기울기 계산 불가)

**Lorenz 곡선 계산 (30~33번줄)**

```r
flows <- outflow_totals[remaining]      # 아직 분류 안 된 구역들의 유출량
ord   <- order(flows)                   # 유출량 오름차순 정렬 인덱스
cum_n <- seq_along(ord) / length(ord)  # 누적 구역 비율 (0~1)
cum_f <- cumsum(flows[ord]) / sum(flows[ord])  # 누적 유출량 비율 (0~1)
```

`order(flows)`: 유출량이 작은 것부터 순서대로 정렬한 인덱스를 반환.  
예: `flows = c(50, 200, 10)` → `order(flows) = c(3, 1, 2)` (가장 작은 게 3번째)

`cumsum()`: 누적합. `cumsum(c(1,2,3))` → `c(1, 3, 6)`

이 두 벡터 (`cum_n`, `cum_f`)가 Lorenz 곡선의 x축, y축입니다.

**마지막 기울기 계산 (35~37번줄)**

```r
nr    <- length(ord)
slope <- (cum_f[nr] - cum_f[nr - 1L]) / (cum_n[nr] - cum_n[nr - 1L])
if (slope <= 1) break
```

Lorenz 곡선의 **마지막 구간 기울기**를 계산합니다.  
기울기 = "마지막 구역이 전체 이동량에서 차지하는 비중 / 전체 구역 수에서 차지하는 비중"

- 기울기 > 1: 마지막(상위) 구역들이 불균형적으로 많은 이동량을 가짐 → 상위 계층 분리 필요
- 기울기 ≤ 1: 이동량이 비교적 균등 → 더 분리할 필요 없음, 루프 종료

**전환점 계산 및 상위 구역 분류 (39~46번줄)**

```r
x_thr     <- 1 - 1 / slope                              # Lorenz 곡선 기울기=1 전환점
cum_rank  <- rank(flows, ties.method = "first") / length(flows)  # 각 구역의 순위 비율
hot_local <- which(cum_rank > x_thr)                    # 전환점 초과 구역들
```

`x_thr`: 기울기가 1이 되는 순위 비율. 이 값보다 순위가 높은 구역들이 "이번 계층"에 해당.

`rank()`: 유출량 순위 부여. `ties.method = "first"`: 같은 값이 있으면 먼저 나온 것이 낮은 순위.

`which()`: 조건이 TRUE인 인덱스 반환. `which(c(F,T,F,T))` → `c(2,4)`

```r
h[remaining[hot_local]] <- level   # 해당 구역들의 계층을 현재 level로 업데이트
remaining <- remaining[-hot_local] # 분류 완료 구역들을 remaining에서 제거
level     <- level + 1L            # 다음 계층으로
```

`remaining[-hot_local]`: 음수 인덱싱. 해당 위치를 제외한 나머지 반환.  
예: `c(1,2,3,4,5)[-c(2,4)]` → `c(1,3,5)`

### 3-2. `.rsg_build_omega()` — ω 가중치 행렬 구성

#### 전체 코드

```r
.rsg_build_omega <- function(phi, h, alpha, beta, max_level) {
  n       <- nrow(phi)              # 54번줄
  phi_adj <- phi                    # 55번줄
  diag(phi_adj) <- 0                # 56번줄

  row_sums  <- rowSums(phi_adj)     # 58번줄
  zero_rows <- row_sums == 0        # 59번줄
  if (any(zero_rows))               # 60번줄
    warning(sprintf("구역 %s의 총 유출량이 0입니다. 균등 가중치로 대체합니다.",
                    paste(which(zero_rows), collapse = ", ")))

  denom <- ifelse(zero_rows, n, row_sums)     # 64번줄
  omega <- phi_adj / denom                    # 65번줄
  if (any(zero_rows)) omega[zero_rows, ] <- 1 / n  # 66번줄

  omega_ii <- alpha + (h - 1L) * (beta - alpha) / (max_level - 1L)  # 68번줄

  for (i in seq_len(n)) {                     # 70번줄
    off_sum <- 1 - omega[i, i]                # 71번줄
    if (off_sum > 0)                          # 72번줄
      omega[i, -i] <- omega[i, -i] / off_sum * (1 - omega_ii[i])
    omega[i, i] <- omega_ii[i]               # 74번줄
  }
  omega
}
```

#### 줄별 설명

**대각 제거 (54~56번줄)**

```r
n       <- nrow(phi)
phi_adj <- phi        # phi를 복사 (원본 보존)
diag(phi_adj) <- 0    # 대각 원소를 0으로 설정
```

`phi_adj <- phi`: R에서 `<-`로 복사하면 새 객체가 생깁니다. `phi_adj`를 수정해도 원본 `phi`는 그대로.  
`diag(phi_adj) <- 0`: 대각 원소 전체를 0으로. `phi_adj[1,1]`, `phi_adj[2,2]`, ... 전부 0이 됨.

**유출량 0 처리 (58~66번줄)**

```r
row_sums  <- rowSums(phi_adj)  # 각 구역의 총 유출량 (대각 제외)
zero_rows <- row_sums == 0     # 유출량이 0인 구역 탐지 (TRUE/FALSE 벡터)
```

`rowSums()`: 행렬 각 행의 합. 결과는 길이 n의 벡터.

```r
if (any(zero_rows))   # 하나라도 0인 구역이 있으면
  warning(...)        # 경고 출력
```

`any()`: 벡터에 TRUE가 하나라도 있으면 TRUE 반환. `all()`은 전부 TRUE일 때만 TRUE.

```r
denom <- ifelse(zero_rows, n, row_sums)     # 0인 행은 분모를 n으로 대체
omega <- phi_adj / denom                    # 행 정규화
if (any(zero_rows)) omega[zero_rows, ] <- 1 / n  # 0인 행은 균등 가중치
```

`phi_adj / denom`: 행렬을 벡터로 나눌 때 R은 자동으로 각 행을 해당 원소로 나눔 (벡터 재활용).

`omega[zero_rows, ]`: `zero_rows`가 TRUE인 행 전체 선택. 해당 행을 `1/n`으로 채워 균등 가중치 부여.

**자기혼합비율 계산 (68번줄)**

```r
omega_ii <- alpha + (h - 1L) * (beta - alpha) / (max_level - 1L)
```

`h`는 길이 n의 정수벡터(각 구역의 계층). 이 수식은 벡터 전체에 한번에 적용됩니다.

- h=1이면: `alpha + 0 = alpha`
- h=max_level이면: `alpha + (max_level-1) * (beta-alpha)/(max_level-1) = beta`

즉, 계층에 따라 alpha~beta 사이를 선형으로 배분합니다.

**대각 재삽입 (70~75번줄)**

```r
for (i in seq_len(n)) {
  off_sum <- 1 - omega[i, i]       # 현재 비대각 합 (= 1 - 대각, 정규화 후라서)
  if (off_sum > 0)
    omega[i, -i] <- omega[i, -i] / off_sum * (1 - omega_ii[i])
  omega[i, i] <- omega_ii[i]       # 대각에 계층 기반 자기혼합비율 삽입
}
```

이 부분만 for 루프를 씁니다. 이유: 각 행마다 비대각 합(off_sum)이 다르고, 그 값으로 비대각 원소를 재조정한 뒤 대각에 새 값을 넣어야 하는데, 이 과정이 행별로 순서 의존적이라 벡터화하기 어렵기 때문입니다.

**재조정 공식 이해:**

조정 전: 비대각 합 = `off_sum`, 합 = 1  
목표: 비대각 합을 `(1 - omega_ii[i])`로 만들고, 대각에 `omega_ii[i]` 삽입 → 행 합 = 1 유지

```r
omega[i, -i] / off_sum         # 비대각을 현재 합으로 나눔 (0~1로 정규화)
             * (1 - omega_ii[i])  # 목표 비대각 합에 맞게 재조정
```

`omega[i, -i]`: i번째 행에서 i번째 열(대각)을 제외한 모든 원소. R의 음수 인덱싱.

### 3-3. `RSG_FSxPy()` — 메인 함수

#### 전체 코드

```r
RSG_FSxPy <- function(data = NULL, group_a, group_b, total, od_matrix,
                      alpha = 0.3, beta = 1.0, max_level = 6L,
                      hierarchy = NULL, verbose = FALSE) {

  pop_x  <- .rsg_resolve_pop(data, group_a)   # 92번줄
  pop_y  <- .rsg_resolve_pop(data, group_b)
  pop_t  <- .rsg_resolve_pop(data, total)
  n      <- .rsg_validate_lengths(pop_x, pop_y, pop_t)

  Total_X <- sum(pop_x, na.rm = TRUE)         # 97번줄
  if (Total_X == 0) return(invisible(list(index = NA_real_, summary = "기준 집단 인구 0")))

  phi <- .rsg_validate_od(od_matrix, n)        # 100번줄

  if (!is.numeric(alpha) || !is.numeric(beta) || alpha < 0 || beta > 1 || alpha > beta)  # 102번줄
    stop("alpha, beta는 0 ≤ alpha ≤ beta ≤ 1 범위여야 합니다.")

  max_level <- as.integer(max_level)           # 105번줄
  if (max_level < 2L) stop("max_level은 2 이상이어야 합니다.")

  if (is.null(hierarchy)) {                    # 108번줄
    phi_nodiag <- phi; diag(phi_nodiag) <- 0
    hierarchy  <- .rsg_lorenz_hierarchy(rowSums(phi_nodiag), max_level)
  } else {
    hierarchy <- as.integer(hierarchy)
    if (length(hierarchy) != n) stop("hierarchy 길이가 구역 수와 다릅니다.")
  }

  omega   <- .rsg_build_omega(phi, hierarchy, alpha, beta, max_level)  # 116번줄
  mixed_y <- .rsg_local_intensity(omega, pop_y)   # 117번줄
  mixed_t <- .rsg_local_intensity(omega, pop_t)   # 118번줄

  valid   <- mixed_t > 0                           # 120번줄
  local_v <- rep(NA_real_, n)                      # 121번줄
  local_v[valid] <- (pop_x[valid] / Total_X) * (mixed_y[valid] / mixed_t[valid])  # 122번줄

  fsxpy_val <- sum(local_v, na.rm = TRUE)          # 124번줄

  if (verbose) cat(sprintf("FSxPy: %.4f\n", fsxpy_val))

  invisible(list(index = fsxpy_val, local = local_v))
}
```

#### 줄별 설명

**파라미터 이름이 PCstar/E_Activity와 다름 (92번줄)**

```r
pop_x <- .rsg_resolve_pop(data, group_a)
pop_y <- .rsg_resolve_pop(data, group_b)
pop_t <- .rsg_resolve_pop(data, total)
```

PCstar는 `a_pop`, `b_pop`, `t_pop`으로 명명했지만 FSxPy는 `pop_x`, `pop_y`, `pop_t`를 씁니다.  
이유: 논문(Liu et al. 2021)이 집단을 x, y로 표기함. 수식과 변수명을 맞춘 것.

**alpha, beta 유효성 검사 (102번줄)**

```r
if (!is.numeric(alpha) || !is.numeric(beta) || alpha < 0 || beta > 1 || alpha > beta)
  stop("...")
```

`||`: OR 연산자. 하나라도 참이면 전체 TRUE → stop() 실행.  
`!is.numeric()`: 숫자가 아니면 TRUE.

PCstar나 E_Activity에는 이런 검사가 없습니다. alpha/beta는 수학적으로 0~1 사이여야만 의미가 있는 파라미터라 범위 검사가 필수입니다.

**max_level 정수 변환 (105번줄)**

```r
max_level <- as.integer(max_level)
```

사용자가 `max_level = 6`(실수)로 넣어도 내부에서 강제로 정수로 변환합니다.  
이후 `h - 1L`, `level + 1L` 등의 정수 연산에서 타입 불일치 오류를 방지합니다.

**hierarchy 분기 (108~114번줄)**

```r
if (is.null(hierarchy)) {
  phi_nodiag <- phi; diag(phi_nodiag) <- 0
  hierarchy  <- .rsg_lorenz_hierarchy(rowSums(phi_nodiag), max_level)
} else {
  hierarchy <- as.integer(hierarchy)
  if (length(hierarchy) != n) stop("hierarchy 길이가 구역 수와 다릅니다.")
}
```

`hierarchy`가 NULL(기본값)이면 OD 유출량으로 자동 계산.  
직접 입력하면 그대로 사용 — 단, 정수 변환과 길이 검사는 필수.

한 줄에 두 문장 (`phi_nodiag <- phi; diag(phi_nodiag) <- 0`): `;`으로 구분하면 같은 줄에 여러 문장 가능. Lorenz 계산용으로 대각을 제거한 임시 행렬을 만드는 것.

**핵심 계산 (116~118번줄)**

```r
omega   <- .rsg_build_omega(phi, hierarchy, alpha, beta, max_level)
mixed_y <- .rsg_local_intensity(omega, pop_y)
mixed_t <- .rsg_local_intensity(omega, pop_t)
```

E_Activity와 동일한 구조지만, W 대신 ω를 씁니다.  
ω는 대각이 양수(자기 구역 포함)라는 점이 핵심 차이.

- `mixed_y[i]`: 구역i 주민이 활동공간에서 만나는 집단Y의 가중합
- `mixed_t[i]`: 구역i 주민이 활동공간에서 만나는 전체인구 가중합

**local_v 계산 방식 (120~122번줄)**

```r
valid   <- mixed_t > 0
local_v <- rep(NA_real_, n)
local_v[valid] <- (pop_x[valid] / Total_X) * (mixed_y[valid] / mixed_t[valid])
```

E_Activity는 `ifelse()`로 한 줄 처리했지만, FSxPy는 3줄로 처리합니다.

이유: `local_v[valid] <- ...`를 쓰면 유효하지 않은(mixed_t = 0) 구역은 **NA로 남습니다**. E_Activity의 `ifelse(L_t > 0, ..., 0)`은 0으로 채웠는데, FSxPy는 NA를 남겨서 "이 구역은 계산 불가"임을 명시적으로 표시합니다. 이후 `sum(local_v, na.rm = TRUE)`로 NA를 제외하고 합산.

**왜 E_Activity와 처리 방식이 다른가?**

FSxPy의 `local_v`는 **인구 가중치가 이미 곱해진** 값 (`pop_x / Total_X` 포함)이라 의미가 다릅니다. 0으로 채우면 "이 구역은 전역 지수에 기여하지 않는다"는 뜻이 되는데, 계산 불가인 구역은 그렇게 처리하기보다 NA로 남겨두는 게 통계적으로 더 정직합니다.
