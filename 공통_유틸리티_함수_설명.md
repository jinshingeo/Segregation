# RSG 패키지 공통 유틸리티 함수 설명

RSG 패키지는 14개의 분리지수 측도 함수들이 공통으로 사용하는 내부 헬퍼 함수들을 3개의 util 파일로 분리해서 관리합니다.  
이 파일들의 함수들은 외부에 직접 노출되지 않으며 (이름이 `.`으로 시작), 각 측도 함수 내부에서 호출됩니다.

---

## 파일 구조 한눈에 보기

```
R/
├── RSG_utils_input.R    ← 입력값 파싱 및 검증
├── RSG_utils_data.R     ← 예제 데이터 로딩 및 그리드 생성
├── RSG_utils_spatial.R  ← 공간 연산 (거리, 가중치, 정규화)
└── RSG_FSxPy.R          ← FSxPy 전용 내부 헬퍼 (Lorenz, ω 행렬)
```

---

## 1. RSG_utils_input.R — 입력값 파싱 및 검증

모든 측도 함수가 맨 처음에 반드시 호출하는 함수들입니다.  
사용자 입력이 열 이름(문자열)이든 숫자 벡터든 관계없이 내부에서 동일하게 처리할 수 있게 해주는 역할입니다.

---

### `.rsg_resolve_pop(data, col)`

**역할:** 사용자가 입력한 `group_a`, `group_b`, `total` 인자를 순수한 숫자 벡터로 변환합니다.

```r
.rsg_resolve_pop <- function(data, col)
```

#### 처리 로직

```
col이 숫자 벡터?  → 그대로 반환
col이 문자열?     → data에서 해당 열을 추출해서 반환
  └─ sf 객체?     → st_drop_geometry()로 geometry 열 제거 후 추출
그 외?            → 에러 발생
```

#### 왜 필요한가?

RSG 패키지의 모든 측도 함수는 두 가지 방식으로 입력을 받을 수 있도록 설계됐습니다:

```r
# 방식 1: sf 객체 + 열 이름 (문자열)
RSG_E_Activity(data = my_sf, group_a = "minority", ...)

# 방식 2: 숫자 벡터 직접 입력
RSG_E_Activity(data = NULL, group_a = c(100, 200, 150, ...), ...)
```

이 두 방식을 각 측도 함수마다 따로 처리하면 코드가 중복됩니다. `.rsg_resolve_pop()`이 이 분기를 한 곳에서 처리해서 측도 함수들은 "숫자 벡터가 들어온다"고만 가정하면 됩니다.

#### 주의: sf 객체에서 geometry 열 제거

sf 객체는 일반 데이터프레임에 geometry 열이 붙은 구조입니다. `data[[col]]`로 직접 꺼내면 geometry 정보가 섞여서 오류가 날 수 있습니다. `sf::st_drop_geometry()`로 geometry를 먼저 제거하고 데이터만 추출합니다.

---

### `.rsg_validate_lengths(...)`

**역할:** 여러 벡터의 길이가 모두 같은지 확인하고, 구역 수 `n`을 반환합니다.

```r
.rsg_validate_lengths <- function(...)
```

#### 사용 예시

```r
# 측도 함수 내부에서
n <- .rsg_validate_lengths(a_pop, b_pop, t_pop)
# → a_pop, b_pop, t_pop 셋이 모두 같은 길이인지 확인
# → 같으면 그 길이(= 구역 수 n)를 반환
# → 다르면 에러 발생
```

#### 왜 필요한가?

사용자가 실수로 구역 수가 다른 데이터를 섞어 입력했을 때 조기에 에러를 잡기 위해서입니다. 이 검증 없이 계산을 진행하면 행렬 연산에서 이상한 결과가 나오거나 나중에 알기 어려운 위치에서 오류가 터집니다.

---

### `.rsg_zone_names(data, n, zone_names)`

**역할:** 구역 이름 벡터를 반환합니다. 사용자가 입력하지 않으면 자동 생성합니다.

```r
.rsg_zone_names <- function(data, n, zone_names = NULL)
```

#### 처리 로직

```
zone_names가 있음? → 그대로 반환
zone_names가 NULL? → "Zone_1", "Zone_2", ..., "Zone_n" 자동 생성
```

#### 어디서 쓰이나?

반환값에 이름이 붙어야 하는 측도에서 사용됩니다. 예를 들어 `RSG_Clustering`은 LQ 벡터와 l 벡터에 구역 이름을 붙여서 반환합니다.

---

### `.rsg_validate_od(od_matrix, n)`

**역할:** OD 행렬이 올바른 형식인지 검증하고, matrix 타입으로 변환해서 반환합니다.

```r
.rsg_validate_od <- function(od_matrix, n)
```

#### 검증 항목

| 검증 | 통과 조건 | 실패 시 |
|------|-----------|---------|
| 크기 | n × n 정방행렬 | 에러 + 실제 크기 출력 |
| 음수 | 모든 원소 ≥ 0 | 에러 |
| 타입 | matrix로 강제 변환 | (데이터프레임 등도 허용) |

## 2. RSG_utils_data.R — 예제 데이터 및 그리드 생성

예제 실행과 테스트를 위한 데이터 준비 함수들입니다.  
측도 계산 자체와는 무관하며, 주로 `examples/` 폴더의 예제 스크립트에서 사용됩니다.

---

### `.rsg_load_segdata(path)`

**역할:** `segdata.rda` 파일을 불러와서 `segdata` 객체를 반환합니다.

```r
.rsg_load_segdata <- function(path)
```

#### 처리 방식

```r
env <- new.env()         # 격리된 환경 생성
load(path, envir = env)  # 그 환경에 파일 로드
env$segdata              # segdata 객체만 추출해서 반환
```

`new.env()`로 격리 환경을 쓰는 이유: `load()`는 기본적으로 현재 환경(`.GlobalEnv`)에 객체를 올립니다. 예상치 못한 변수명 충돌을 막기 위해 격리된 임시 환경에 로드하고 필요한 객체만 꺼냅니다.

---

### `.rsg_make_grid_sf(n)`

**역할:** n×n 격자 형태의 sf 폴리곤 객체를 생성합니다.

```r
.rsg_make_grid_sf <- function(n)
```

#### 동작 설명

```
[0,n] × [0,n] 범위의 정사각형 영역을 n×n 격자로 분할
→ 각 셀이 하나의 구역(폴리곤)이 되는 sf 객체 반환
→ grid_id 열: 1 ~ n²
```

#### 왜 필요한가?

`segdata.rda`는 100개 구역의 인구 데이터(숫자 행렬)만 담고 있고, geometry 정보가 없습니다. sf 기반 측도(`RSG_Clustering` 등)를 실행하려면 각 구역에 공간 좌표(폴리곤)가 있어야 합니다. `.rsg_make_grid_sf(10)`이 10×10 격자 sf 객체를 만들어주면, 그 위에 인구 데이터를 붙일 수 있습니다.

---

### `.rsg_segdata_to_sf(segdata, col_indices, col_names)`

**역할:** segdata 행렬의 특정 열을 골라서 sf 그리드 객체로 변환합니다.

```r
.rsg_segdata_to_sf <- function(segdata, col_indices = NULL, col_names = NULL)
```

#### 처리 흐름

```
1. segdata 행 수의 제곱근 → n (10×10 그리드 가정)
2. .rsg_make_grid_sf(n) 호출 → 빈 그리드 sf 생성
3. col_indices로 원하는 열 선택 (NULL이면 전체)
4. col_names로 열 이름 지정 (NULL이면 원래 이름 유지)
5. grid_id 기준으로 merge → geometry가 붙은 sf 반환
```

#### 사용 예시

```r
# segdata의 1~2번 열(패턴1)을 minority/majority 이름으로 sf로 변환
sf_p1 <- .rsg_segdata_to_sf(
  segdata,
  col_indices = c(1, 2),
  col_names   = c("minority", "majority")
)
```

---

### `.rsg_grid_dist_matrix(n)`

**역할:** n×n 격자의 구역 간 유클리드 거리 행렬을 생성합니다.

```r
.rsg_grid_dist_matrix <- function(n)
```

#### 동작 설명

```r
xy <- expand.grid(x = 1:n, y = 1:n)  # n² 개 격자점의 (x, y) 좌표
dist(xy)                               # 모든 점 쌍의 유클리드 거리
```

sf 객체 없이도 격자 구조라면 거리 행렬을 바로 만들 수 있는 단순 유틸입니다.  
`.rsg_dist_matrix()`의 sf 경로 대신 사용할 수 있는 경량 대안입니다.

## 3. RSG_utils_spatial.R — 공간 연산 헬퍼

거리 계산, 가중치 행렬 생성, OD 정규화 등 **측도 계산의 핵심 수식**을 담당하는 함수들입니다.  
Gaussian 커널 기반 측도(`RSG_D_Global`, `RSG_I_Global` 등)와 OD 기반 측도(`RSG_E_Activity`, `RSG_FSxPy`)가 모두 이 파일에 의존합니다.

---

### `.rsg_get_coords(x)`

**역할:** sf 객체 또는 좌표 행렬에서 구역 중심점 좌표 (x, y)를 추출합니다.

```r
.rsg_get_coords <- function(x)
```

#### 처리 로직

```
입력이 sf 객체?
  └─ POINT/MULTIPOINT 타입? → st_coordinates()로 좌표 직접 추출
  └─ POLYGON 등?            → st_centroid()로 중심점 계산 후 좌표 추출
입력이 행렬/데이터프레임?   → 첫 번째, 두 번째 열을 x, y로 해석
```

폴리곤 데이터에서 `st_centroid()` 호출 시 `suppressWarnings()`를 쓰는 이유: 투영 좌표계가 아닌 경위도 좌표계에서 centroid 계산 시 "경위도 좌표에서 centroid는 부정확할 수 있음"이라는 경고가 나오는데, 예제 수준에서는 무시해도 되므로 억제합니다.

---

### `.rsg_dist_matrix(data, dist_matrix, n)`

**역할:** 거리 행렬을 반환합니다. 이미 행렬이 있으면 그대로 쓰고, 없으면 data에서 계산합니다.

```r
.rsg_dist_matrix <- function(data, dist_matrix, n)
```

#### 처리 로직

```
dist_matrix가 있음? → 그대로 반환 (검증 없이 신뢰)
dist_matrix가 NULL?
  └─ data가 있음?   → .rsg_get_coords(data)로 좌표 추출 → dist() 계산
  └─ data도 NULL?   → 에러 발생
```

#### 측도 함수에서의 사용 패턴

```r
# 사용자가 dist_matrix를 직접 줬으면 그걸 쓰고,
# 안 줬으면 sf 객체에서 자동 계산
d_mat <- .rsg_dist_matrix(data, dist_matrix, n)
```

이 패턴 덕분에 모든 거리 기반 측도 함수들이 두 가지 입력 방식을 지원합니다.

---

### `.rsg_gaussian_weights(dist_mat, bw)`

**역할:** 거리 행렬을 Gaussian 커널 가중치 행렬로 변환하고 행 정규화합니다.

```r
.rsg_gaussian_weights <- function(dist_mat, bw)
```

#### 수식

```
W[i,j] = exp(-d[i,j]² / (2 × bw²))
W       = W / rowSums(W)    ← 각 행의 합이 1이 되도록 정규화
```

#### 파라미터

- `bw` (bandwidth): 커널 반경. 클수록 먼 구역에도 가중치 부여. 작을수록 가까운 구역만 강조.
- 대각(자기 자신)도 포함됨: `d[i,i] = 0`이므로 `exp(0) = 1`로 자기 구역이 가장 큰 가중치를 가짐

#### 사용하는 측도

`RSG_D_Global`, `RSG_D_Local`, `RSG_I_Global`, `RSG_I_Local`, `RSG_NSI` 등 Gaussian 커널 기반 측도들

---

### `.rsg_local_intensity(W, pop)`

**역할:** 가중치 행렬 W와 인구 벡터 pop의 행렬 곱을 계산합니다.

```r
.rsg_local_intensity <- function(W, pop)
```

#### 수식

```
result[i] = Σ_j W[i,j] × pop[j]
```

즉, "구역 i 주변의 pop 가중합"을 모든 구역에 대해 한번에 계산합니다.

#### 왜 이걸 별도 함수로 만들었나?

```r
# 이 패턴이 여러 측도에서 반복됨
L_a <- W %*% a_pop   # 구역i 주변의 집단A 가중합
L_t <- W %*% t_pop   # 구역i 주변의 전체인구 가중합
```

`as.vector()`로 감싸는 이유: `W %*% pop`은 행렬 곱 결과로 n×1 행렬이 나옵니다. `as.vector()`로 길이 n의 벡터로 변환해야 이후 연산이 자연스럽습니다.

---

### `.rsg_normalize_od(phi)`

**역할:** OD 행렬의 대각을 0으로 만들고, 행 합이 1이 되도록 정규화합니다.

```r
.rsg_normalize_od <- function(phi)
```

#### 처리 흐름

```
1. diag(phi) <- 0               ← 자기 구역 이동 제거
2. row_sums <- rowSums(phi)
3. row_sums == 0인 행 탐지      ← 아무 데도 이동 않는 구역
   → 경고 출력
   → row_sums를 n으로 대체, 해당 행을 1로 채움 (균등 가중치)
4. phi / row_sums               ← 행 정규화
```

#### 결과 행렬의 의미

```
W[i,j] = (구역i → 구역j 이동량) / (구역i의 총 유출량)
       = "구역i 주민이 이동할 때 구역j로 갈 확률"
```

대각이 0이므로 "자기 구역에서 자기 구역 만남"은 제외됩니다.  
→ `RSG_E_Activity`에서 최종 W는 대각 = 0  
→ `RSG_FSxPy`는 이 함수를 직접 안 쓰고, 대신 `.rsg_build_omega()`가 동일 로직에 대각 재삽입을 추가함

## 4. RSG_FSxPy.R 내부 헬퍼 — FSxPy 전용

이 두 함수는 `RSG_utils_*`에 포함되지 않고 `RSG_FSxPy.R` 안에 정의돼 있습니다.  
FSxPy에서만 쓰이는 전용 로직이기 때문입니다.

---

### `.rsg_lorenz_hierarchy(outflow_totals, max_level)`

**역할:** 각 구역의 총 유출량을 기반으로 Lorenz 곡선 알고리즘을 적용해 계층(h)을 자동 분류합니다.

```r
.rsg_lorenz_hierarchy <- function(outflow_totals, max_level = 6L)
```

#### 입력/출력

- 입력: 각 구역의 총 유출량 벡터 (대각 제외한 OD 행 합계)
- 출력: 각 구역의 계층 번호 벡터 (1 = 가장 활동적, max_level = 가장 비활동적)

#### 알고리즘 단계별 설명

```
초기화: 모든 구역 h = max_level (일단 최하위 계층으로)

반복 (level = 1, 2, ..., max_level-1):
  1. 아직 분류 안 된(remaining) 구역들의 Lorenz 곡선 마지막 기울기 계산
     slope = Δ누적유출량 / Δ누적구역수  (마지막 구간)

  2. slope ≤ 1 이면 중단 → 나머지는 전부 현재 max_level

  3. slope > 1 이면:
     x_thr = 1 - 1/slope  (Lorenz 곡선에서 기울기=1인 전환점)
     누적순위가 x_thr 초과인 구역들 → 현재 level로 분류
     남은 구역에서 제거 후 다음 level로 진행
```

#### 직관적 이해

Lorenz 곡선의 기울기가 1보다 크다 = "상위 일부 구역이 전체 이동량에서 불균형적으로 큰 비중을 차지한다" = 이동량 불평등이 존재한다. 이 구역들을 "상위 계층"으로 분류하고, 다음 반복에서 나머지를 같은 방식으로 다시 분류합니다.

#### 예시

```
유출량: c(1000, 800, 200, 50, 30)  max_level = 6

→ 1000, 800이 불균형 → level 1 분류
→ 남은 200, 50, 30 중 200이 불균형 → level 2 분류
→ 남은 50, 30은 기울기 ≤ 1 → 전부 max_level(6)

결과: h = c(1, 1, 2, 6, 6)
```

---

### `.rsg_build_omega(phi, h, alpha, beta, max_level)`

**역할:** OD 행렬과 계층 벡터를 받아 FSxPy의 최종 가중치 행렬 ω를 구성합니다.

```r
.rsg_build_omega <- function(phi, h, alpha, beta, max_level)
```

#### 처리 흐름

```
1단계: 대각 제거 + 행 정규화 (← .rsg_normalize_od 와 동일 시작점)
       diag(phi_adj) <- 0
       omega <- phi_adj / rowSums(phi_adj)

2단계: 각 구역의 자기혼합비율 계산
       omega_ii[i] = alpha + (h[i] - 1) × (beta - alpha) / (max_level - 1)

3단계: 각 구역 i에 대해:
       비대각 원소 재조정 → omega[i, -i] / 합계 × (1 - omega_ii[i])
       대각에 omega_ii 삽입 → omega[i, i] = omega_ii[i]
```

#### omega_ii 수식의 의미

| 계층 h | omega_ii (alpha=0.3, beta=1.0, max_level=6) |
|--------|------|
| 1 (가장 활동적) | 0.3 + 0 × 0.14 = **0.30** |
| 2 | 0.3 + 1 × 0.14 = **0.44** |
| 3 | 0.3 + 2 × 0.14 = **0.58** |
| 4 | 0.3 + 3 × 0.14 = **0.72** |
| 5 | 0.3 + 4 × 0.14 = **0.86** |
| 6 (가장 비활동적) | 0.3 + 5 × 0.14 = **1.00** |

→ 활동적일수록 자기 구역 체류 30%, 비활동적일수록 100%까지 선형 증가

#### 최종 ω 행렬의 특성

- 각 행의 합 = 1 (대각 포함)
- 대각 원소 = alpha ~ beta 범위의 양수
- 비대각 원소 = OD 비율에 비례하되 (1 - omega_ii) 범위 내로 조정
